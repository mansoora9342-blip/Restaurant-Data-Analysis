# Restaurant Data Analysis Project

This project focuses on analyzing a real-world restaurant dataset containing 7,000+ records to understand customer preferences, pricing behavior, service availability, and location-based performance.

# Problem Statement

Raw restaurant data is often messy and inconsistent, making it difficult for businesses to identify trends. The goal was to clean and structure the data, uncover meaningful patterns, and convert raw information into actionable insights for decision-making.

# Technical Approach

- Cleaned and standardized messy data (country codes, city names, delivery flags, cuisines)

- Transformed multi-value cuisine fields into analyzable formats

- Performed exploratory data analysis using Python (Pandas, NumPy, Matplotlib, Seaborn)

- Built metrics for ratings, votes, price ranges, and service availability

# Key Insights & Patterns

- Identified the most popular cuisines and cuisine combinations

- Found cities with high restaurant density and highest average ratings

- Discovered that higher price ranges are more likely to offer online delivery and table booking

- Observed a positive relationship between customer votes and ratings

- Detected geographic clusters around major urban areas

# Unique Value

Instead of only reporting counts, this project connects pricing, customer engagement, and service availability to explain why certain restaurants perform better.

# Outcome

The analysis provides clear insights that can help restaurant owners and platforms optimize pricing, improve service offerings, and target high-performing locations.
